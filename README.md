### 📌 개요

- **구현 기간**: 2026.03.12
- **기술 스택**: Python

> 프로젝트 별 git commit 메시지를 수집해 비슷한 내용끼리 묶어 문서로 반환하는 자동화 어플리케이션

### 🎯 목적

프로젝트를 진행하다 보면 여러 저장소에서 다양한 기능 개발과 수정 작업이 반복적으로 이루어집니다.
이러한 작업들은 대부분 Git 커밋 메시지 형태로 기록되지만 시간이 지나면 어떤 기능을 개발했는지 전체 흐름을 다시 파악하기 어려워지곤 합니다.

특히 취업 준비나 포트폴리오 정리를 위해 프로젝트 경험을 문서화할 때 여러 저장소의 커밋 기록을 다시 확인하며 어떤 기능을 구현했고 어떤 작업을 수행했는지 정리하는 데에는 많은 시간이 소요됩니다.

본 프로젝트는 이러한 문제를 해결하기 위해 Git 커밋 로그를 기반으로 개발 작업을 자동으로 정리하고 문서화할 수 있는 도구를 제안합니다. 개발 작업 이력을 구조적으로 정리하고 포트폴리오나 기술 문서 작성 시 활용할 수 있는 기반 데이터를 자동으로 생성할 수 있도록 하는 것을 목표로 합니다.

### ⚙️ 구조

본 애플리케이션은 여러 프로젝트 저장소를 대상으로 Git 커밋 로그를 수집한 뒤,
다음과 같은 처리 과정을 통해 개발 작업 문서를 자동으로 생성합니다.

#### Git 저장소 탐색 및 커밋 로그 수집

지정된 workspace 경로 내의 Git 저장소들을 탐색하여 git log 명령을 통해 커밋 메시지를 수집합니다. 수집된 커밋 기록은 저장소별로 분리하여 commits 디렉토리에 저장됩니다.

#### 메시지 전처리

수집된 커밋 메시지를 분석하여 다음과 같은 전처리를 수행합니다.

1. Merge 커밋 제거
2. feat, fix, refactor 등의 커밋 prefix 제거
3. 메시지 구조 정리

이를 통해 분석에 불필요한 노이즈를 제거합니다.

#### 기능 기반 1차 그룹화

커밋 메시지 내에 `>` 구분자가 존재하는 경우
가장 앞의 경로를 기준으로 기능 단위 그룹을 생성합니다.

```
Auth > Login > 로그인 API 구현
Auth > Login > 토큰 검증 로직 구현
```

위 커밋들은 Auth 기능 그룹으로 묶입니다.

#### 소규모 그룹 클러스터링

특정 기능 그룹의 커밋 개수가 일정 기준(5개) 이하일 경우 의미적으로 유사한 작업을 자동으로 묶기 위해 문장 임베딩 기반 클러스터링을 수행합니다.

- 커밋 메시지를 문장 단위로 정리
- Sentence Transformer 모델을 이용해 문장 임베딩 생성
- 생성된 임베딩 벡터를 기반으로 K-Means 클러스터링 수행

초기 구현에서는 밀도 기반 클러스터링 알고리즘인 DBSCAN을 사용했으나 기능 경로 기반으로 1차 그룹화가 된 이후에는 각 그룹의 데이터 수가 많지 않아 불필요하게 많은 클러스터가 생성되거나 결과를 제어하기 어려운 문제가 있었습니다. 특히 DBSCAN은 클러스터 개수를 직접 지정할 수 없기 때문에 원하는 수준의 그룹화를 만들기 어려웠기에 이를 보완할 수 있는 K-Means 알고리즘을 사용했습니다.

#### Markdown 보고서 생성

정리된 결과를 기반으로 저장소별 Markdown 문서를 생성합니다. 생성된 문서는 reports 디렉토리에 저장되며, 각 프로젝트의 주요 작업 내용을 다음과 같은 구조로 확인할 수 있습니다.

```
# repository-name

## Auth
- 로그인 API 구현
- 로그인 토큰 검증

## cluster-0
- 채팅 웹소켓 연결
- 메시지 전송 로직 구현
```

### ✅ 성과

여러 프로젝트에서 수행했던 개발 작업을 다시 확인하고 정리하는 과정을 자동화하여
개발 이력을 체계적으로 문서화할 수 있는 환경을 구축했습니다.

- 여러 저장소의 커밋 기록을 한 번에 정리 가능
- 기능 단위로 개발 작업을 자동 분류
- 반복적인 포트폴리오 정리 작업 시간 단축
- Markdown 기반 문서로 바로 활용 가능

이를 통해 개발 경험을 단순한 커밋 로그가 아닌 구조화된 개발 작업 기록 형태로 관리할 수 있도록 개선했습니다.

### 👀 확장 계획

현재 애플리케이션은 Git 커밋 메시지를 기반으로 기능 그룹화와 클러스터링을 수행하여 개발 작업을 구조적으로 정리하는 기능을 제공합니다. 그러나 작은 기능 단위로 메시지가 작성되어 있기 때문에 단순히 나열된 목록만으로는 실제 수행한 작업의 맥락을 완전히 파악하기 어렵습니다.

이를 보완하기 위해 LLM 기반 요약 기능을 추가하고자 합니다.

- 클러스터링된 커밋 메시지들을 LLM을 통해 하나의 작업 흐름으로 요약
- 각 기능 단위에서 수행된 핵심 작업을 bullet 형식으로 정리

이러한 기능을 통해 개발 경험을 자연어 기반의 작업 기록으로 자동 정리하는 도구로 발전시키는 것을 목표로 하고 있습니다.
